# Plan 2: Multiplayer

## Scope

Add real-time multiplayer support so two or more players can compete against each other over WebSockets. This covers the full stack: server infrastructure, networking protocol, lobby system, server-authoritative game state, garbage mechanics, client-side prediction, and all multiplayer UI (lobby, opponent boards, garbage indicator, game-over/rematch).

## Goals

1. **WebSocket server** — Node.js server handling real-time communication between players
2. **Room/lobby system** — Create, join, and spectate game rooms with room codes
3. **Server-authoritative game state** — Server runs the game engine as the source of truth; clients predict locally for responsiveness
4. **Garbage mechanics** — Standard multiplayer garbage rules: line clears send garbage rows, with combo/T-spin/back-to-back bonuses and a delay queue
5. **Multiplayer UI** — Lobby, opponent board previews, garbage queue indicator, win/loss screen, rematch flow
6. **Resilient networking** — Client-side prediction, disconnect handling (reconnect window or forfeit), latency display

## Key Design Decisions

- **Server authority**: The game engine from Plan 1 runs on the server as the authoritative state. Clients send inputs (move, rotate, drop), server validates and broadcasts results. Clients run their own engine instance for prediction/rendering.
- **Room-based model**: Simple room system with room codes for private games and optional quick-play for public matchmaking. Each room holds 2+ players and a shared game session.
- **Shared TypeScript types**: Network protocol message types are shared between client and server to ensure type safety.
- **Engine reuse**: Plan 1's engine is designed to be browser-independent, so it runs unchanged on the server.
- **Garbage queue with delay**: Incoming garbage is queued with a configurable delay before insertion (like tetr.io), giving the receiving player time to counteract.

## Constraints

- The game engine, scoring, and renderer from Plan 1 must be complete before certain PRs can begin (noted in dependencies below).
- The lobby/networking infrastructure has no Plan 1 dependency and can be built in parallel.

### PR: Garbage network integration
- **description**: Wire garbage mechanics into the multiplayer server. When a player clears lines, the server calculates garbage using the garbage engine and distributes it to opponents. Server broadcasts garbage queue state so clients can display pending garbage. Default targeting is even-split across all opponents. ### PR: Garbage targeting strategies and attack power
- **description**: Implement configurable garbage targeting for multiplayer games. Room settings define which targeting strategies are available (any combination of: random, attackers, KOs, manual). Each player selects their active strategy, changeable mid-game. **Random**: garbage sent to a random opponent (default). **Attackers**: automatically targets players currently targeting you. **KOs**: targets the player closest to topping out. **Manual**: player clicks an opponent board to select a specific target. Includes a per-player attack power multiplier earned through KOs (1 KO → 1.25x, 2 → 1.5x, 4 → 1.75x, 6+ → 2x), applied to outgoing garbage at distribution time. **Human testing needed**: verify strategy switching mid-game, manual target selection by clicking opponent boards, and attack power scaling after KOs across multiple browser tabs.
- **tests**: Each targeting strategy selects correct target(s), attack power multiplier calculation at KO thresholds, multiplier applied to outgoing garbage correctly, strategy switching mid-game, manual target persistence, game settings validation (at least one strategy enabled), attacker strategy retaliation logic, KO strategy targets lowest-health player
- **files**: `server/targeting.ts`, `server/attack-power.ts`, `server/garbage-manager.ts` (extend), `shared/targeting-types.ts`, `shared/protocol.ts` (add targeting messages), `src/ui/TargetingSelector.tsx`, `src/ui/OpponentBoard.tsx` (extend with target selection), `src/net/game-client.ts` (extend)
